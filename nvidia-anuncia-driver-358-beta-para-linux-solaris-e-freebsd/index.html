NVIDIA anuncia driver 358 (beta) para Linux, Solaris e FreeBSD

Douglas Santos em Notícias
13/10/2015 · 1 minuto de leitura

Atenção

Este artigo foi migrado do Wordpress e pode conter erros de formatação.

Caso encontre algum problema deixe um comentário que iremos resolver o mais breve possível.

A NVIDIA disponibilizou a primeira versão pública beta do driver 358 (358.09) para Linux, FreeBSD e Solaris. Construído sobre o driver 355, a série 358 acrescenta mais peças ao quebra-cabeças da interface com DRM/KMS (Direct Rendering Manager/Kernel Mode Setting) e continua melhorando o suporte ao Mir/Wayland.

A versão do driver adicionou um novo módulo nvidia-modeset ao kernel para trabalhar em conjunto com o módulo nvidia kernel. Em uma próxima versão do driver, o nvidia-modeset kernel será usado como “base para a interface de configuração do modo fornecido pelo gerenciador de renderização direta do kernel”, de acordo com o anuncio.

Esta versão, também corrige o desempenho do OpenGL para: alguns servidores gráficos (X servers); vazamento de memória; flickering e delays ao entrar e/ou sair do modo G-SYNC, dentre várias outras correções.

No driver proprietário para Linux, a NVIDIA também adicionou um novo protocolo para extensões GLX, um novo mecanismo de alocação de memória do sistema para grandes alocações no driver OpenGL e suporte às placas GeForce 805A e GTX 960A.
